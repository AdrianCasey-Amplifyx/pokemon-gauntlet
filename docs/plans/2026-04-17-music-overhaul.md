# Music Overhaul — Pokemon Gauntlet

**Date:** 2026-04-17
**Status:** SHIPPED (core bucket) — 14 tracks live on `main` as of 2026-04-17. See §7 for per-bucket state.

## Implementation note (2026-04-17)

The plan originally recommended **Option B** (ship verbatim MIDI transcriptions with attribution). During implementation we pivoted to an **Option A-equivalent "in-the-style-of"** approach: original compositions authored in the same key, tempo, and mood as each RBY reference track. Rationale:

1. Keeps the `CLAUDE.md` "No external assets, procedural Web Audio" design principle strict — everything stays as TypeScript data in `src/audio/tracks.ts`, no MIDI parser, no binary files committed.
2. Sidesteps the legal flag in §5 entirely (compositions are original, not lifted).
3. Deliverable in a single session — data-only additions, no new dependencies.

Tracks land in 2 channels each (square-lead melody + triangle/sawtooth bass), written with standard chord progressions in the target key, so they land squarely in "Game Boy-ish" territory without copying phrases. The per-world / per-scene assignment table in §3 is unchanged.

## 1. Where we are today

All in-game music is generated by `src/audio/MusicManager.ts` using the Web Audio API. There are only **two tracks**:

| Track id   | Where it plays                                   | Composition                                                        |
| ---------- | ------------------------------------------------ | ------------------------------------------------------------------ |
| `"map"`    | `MainMenuScene` (town hub) + `MapScene` (dungeon) | 18-note triangle melody + 8-note sine bass, C major, looped        |
| `"battle"` | `BattleScene`                                     | 24-note square melody + 16-note sawtooth bass, A minor, looped     |

`TitleScene`, `StarterSelectScene`, `WorldSelectScene` currently run **silent** (they all call `MusicManager.stop()`). Boss rooms play the same loop as wild encounters. Every world — Viridian Path through Indigo Plateau — shares the same map music.

The SFX side is in reasonable shape: six one-shot presets (`purchase`, `heal`, `item_use`, `hatch`, `learn`, `error`) already cover most UI moments and can stay.

## 2. What the Pokemon R/B/Y soundtrack actually contains

Junichi Masuda's original score (Game Boy, 1996) — as published on the *Pokémon Red & Green: Super Music Collection* — has **43 distinct pieces**. Nayuki (linked below) has pitch-and-octave-accurate 4-channel transcriptions as MIDI + PDF for every one of them.

Game Boy audio is 4 monophonic channels (2 pulse + 1 triangle-ish wave + 1 noise). Channel 2 usually carries melody, channel 1 carries harmony, channel 3 carries bass. That maps cleanly onto our existing `OscillatorType` palette (`square` / `triangle` / `sawtooth` / `sine`), so we don't need to change audio architecture to host these.

## 3. Proposed assignment to Pokemon Gauntlet

The goal is a **one-to-one mapping from scene/context → RBY track**, with the 8 worlds getting distinct music. Names are chosen so each world's thematic pool lines up with the original location's feel.

### 3.1 Global / UI

| Scene | Track | Rationale |
|---|---|---|
| `BootScene` | *(none)* | Splash is too brief to start audio |
| `TitleScene` | **Title Screen** (C maj, 134 BPM) | Direct lift — it *is* the title screen theme |
| `StarterSelectScene` | **Oak Pokémon Lab** (A maj, 137.8 BPM) | You're literally picking a starter from Oak's lab |
| `MainMenuScene` (town hub) | **Viridian / Pewter / Saffron City** (E maj, 134 BPM) | Our town has mart + center + trader + adventure gate — it's a "big town," and this is the longest/most-featured town theme (32 bars). Pallet Town is too sleepy for our hub |
| `WorldSelectScene` (if re-introduced) | **Guide / Hurry Along** (A maj, 150.7 BPM) | "Follow-me" travel cue |

### 3.2 Per-world map music (dungeon `MapScene`)

The 8 worlds map onto the original locations their names evoke:

| # | World | Track | Why |
|---|---|---|---|
| 0 | Viridian Path | **Route 1** (D maj, 126.9 BPM) | The first route in the real game, paired with Viridian City |
| 1 | Mt. Moon Depths | **Mt. Moon** (120.6 BPM) | Direct match |
| 2 | Cerulean Caves | **Route 24** (E maj, 126.9 BPM) | Route 24 leads north out of Cerulean across Nugget Bridge; our pool is water-leaning but "caves" so the actual Cerulean Cave doesn't have a theme — Route 24 is the closest adjacent piece |
| 3 | Vermilion Docks | **S.S. Anne** (A maj, 150.7 BPM) | S.S. Anne is docked at Vermilion — literally "the docks" |

| 7 | Indigo Plateau | **Victory Road** → **Indigo Plateau** | Early rooms use Victory Road (approach); rooms 20–25 cross-fade to Indigo Plateau (F maj, 146.1 BPM) for the final push |

## 4. Technical approach

### 4.1 Three implementation options

| | Approach | Pros | Cons |
|---|---|---|---|
| **A** | Hand-transcribe each track into the existing `{ note, dur }[]` arrays, one melody + one bass per track | Zero new dependencies; same engine; keeps the "procedural, no external assets" principle strict | ~16–32 bars × 40 tracks is a lot of hand-entry; easy to get wrong; we lose Game Boy's harmony channel |
| **B** | Ship tiny MIDI files (<4 KB each) under `src/audio/tracks/`, play via a lightweight Web Audio MIDI scheduler | Uses Nayuki's already-accurate 4-channel transcriptions verbatim; ~40 tracks × ~3 KB = ~120 KB total; still procedurally synthesized at runtime (no sampled audio) | First time we ship non-code data; need a ~200 LOC MIDI parser + channel → oscillator router |
| **C** | Ship `.ogg` / `.mp3` renders | Highest audio quality; trivial to play with Phaser's audio manager | Breaks the `CLAUDE.md` "no external assets, procedural Web Audio" design principle; ~30–50 KB per track = ~1.5 MB; re-encoding for each variant |

**Recommendation: Option B.** MIDI is notation data, not sampled audio — we still synthesize at runtime, which keeps the spirit of "procedural Web Audio." It lets us take Nayuki's pitch-accurate 4-channel transcriptions and route each channel to an `OscillatorNode` with a matching wave shape (pulse → square, triangle-wave → triangle, noise → filtered sawtooth). ~120 KB total is well under one sprite texture's worth of memory.

### 4.2 MusicManager API changes (Option B sketch)

```ts
// src/audio/MusicManager.ts
export type TrackId =
  | "title" | "lab" | "town" | "guide"
  | "route1" | "mt_moon" | "route24" | "ss_anne"
  | "celadon" | "silph" | "mansion" | "victory_road" | "indigo"
  | "battle_wild" | "battle_gym" | "battle_trainer" | "battle_final"
  | "none";

export type JingleId =
  | "victory_wild" | "victory_gym" | "hall_of_fame"
  | "healed" | "evolution" | "level_up" | "caught" | "item_get";

interface Track {
  channels: Array<{
    wave: OscillatorType;      // square / triangle / sawtooth
    volume: number;
    notes: Array<{ midi: number; dur: number; delay: number }>;
    loopStart: number;         // seconds from track start
  }>;
  bpm: number;
}

class MusicManagerClass {
  play(id: TrackId): void;
  playJingle(id: JingleId, duckMusic?: boolean): Promise<void>;
  // existing:
  stop(): void;
  playSFX(kind: SFXKind): void;
}
```

Tracks live in `src/audio/tracks/<id>.ts` (or `.mid` decoded at boot into the same shape). Each track carries its own BPM so we don't over-normalise.

### 4.3 Scene wiring (call sites)

| File | Current call | New call |
|---|---|---|
| `src/scenes/TitleScene.ts:14` | `MusicManager.stop()` | `MusicManager.play("title")` |
| `src/scenes/StarterSelectScene.ts` | *(none)* | `MusicManager.play("lab")` in `create()` |
| `src/scenes/MainMenuScene.ts:89` | `MusicManager.play("map")` | `MusicManager.play("town")` |
| `src/scenes/MapScene.ts:86` | `MusicManager.play("map")` | `MusicManager.play(trackForWorld(state.activeWorld))` |
| `src/scenes/BattleScene.ts:65` | `MusicManager.play("battle")` | `MusicManager.play(trackForBattle(roomIndex, worldIndex, isBoss))` |
| `src/scenes/BattleScene.ts` victory branch | *(none)* | `MusicManager.playJingle("victory_wild" \| "victory_gym")` |

`trackForWorld` / `trackForBattle` are small helpers placed in `src/data/worlds.ts` so the world table and its music stay co-located.

### 4.4 Performance & memory

- MIDI parse happens once at boot in `PreloadScene`, same place we already bake sprite textures.
- Peak simultaneous oscillators: 4 (Game Boy native) — no change to the current budget.
- No streaming; no audio file I/O.

## 5. Legal / attribution

These compositions are **© 1996 Game Freak / Nintendo**. The nayuki.io transcriptions are published "for educational / archival use." Before we ship this publicly on GitHub:

1. Add an `AUDIO_ATTRIBUTION.md` crediting Junichi Masuda as composer and Nayuki as the transcription source.
2. Decide repo stance: portfolio / non-commercial tribute is generally tolerated; any commercial distribution would require either a Nintendo licence (won't happen) or swapping to original "in-the-style-of" compositions.
3. If we want zero legal exposure: use the same **keys, tempos, chord progressions, and channel layouts** from the table above but write new melodies — still gets us 90% of the "sounds like R/B" feel without lifting the tunes verbatim.

**Recommendation:** ship verbatim transcriptions with attribution for now (the repo is a personal learning project); flag the question in the README so we revisit if we ever monetise.

## 7. Suggested delivery order

If approved, ship in small buckets so each step is testable on its own:

1. **Track schema + loader** ✅ — `TrackId` / `TrackDef` / `TrackChannel` types in `src/audio/tracks.ts`; `MusicManager` refactored to one timer per channel with beats→seconds conversion. Inline `MAP_*` / `BATTLE_*` arrays removed.
2. **Title + Town + Lab** ✅ — `TitleScene` → `"title"`, `MainMenuScene` → `"town"`, `StarterSelectScene` → `"lab"`.
3. **Per-world map music** ✅ — 8 tracks (`world0`…`world7`) behind `trackForWorld(worldIndex)` in `src/data/worlds.ts`.
4. **Battle tiers** ✅ — `battle_wild` / `battle_boss` / `battle_final` behind `trackForBattle(worldIndex, isBoss)`. Simplified from the 4-tier plan (wild / gym / trainer / final) to 3 tiers because rooms 5/10/15/20/25 all share the `isBossBattle` flag and there's no room-level signal in the scene registry to split "gym" from "trainer" further without adding state.
5. **Victory jingles** — NOT SHIPPED in this bucket. Keep existing `playSFX` presets for now; revisit if the victory screen feels flat.
6. **Evolution / level-up / caught jingles** — NOT SHIPPED. The existing `hatch` / `learn` / `purchase` SFX remain; the RBY-style multi-phrase jingles would need a new "play jingle, then resume track" API surface that's larger than this overhaul warranted.
7. **Legal + attribution docs** — N/A after the Option A pivot. All compositions are original; no attribution doc needed beyond the plan's own prose.

Each bucket is one commit, follows the usual CHANGELOG / GAME_DESIGN update pass.
