### 1.2 Move mechanics — full Gen 1 pass (TODO #10, #11) ✅

One-pass implementation of every special-mechanic move currently in `src/data/moves.ts`. Each becomes a typed effect on the `MoveData` / resolved in `battleStateMachine.ts`.

| Mechanic | Moves | Behaviour |
|---|---|---|
| **HP drain** | Absorb, Mega Drain, Leech Life | Heal user for 50% of damage dealt |
| **Leech Seed** | Leech Seed | Grass-type only; each turn: target loses 1/8 max HP, user heals same amount; persists until switch/faint |
| **Bide** | Bide | Lock user for 2 turns storing damage taken; turn 3 unleash 2× stored damage |
| **Rollout** | Rollout | Power doubles each consecutive hit up to 5 turns; resets on miss, switch, or different move |
| **Counter** | Counter | Next-turn resolve: deals 2× the physical damage received this turn (Fighting-type, 0 base power) |
| **Semi-invulnerable turn 1** | Dig, Fly | Turn 1 user is off-screen + untargetable; turn 2 attack lands. Sprite hides during charge turn |
| **Charge turn 1** | Solar Beam, Skull Bash, Sky Attack | Turn 1 charge message, turn 2 attack |
| **OHKO** | Horn Drill, Fissure, Guillotine | Gen 1 formula: hits if user level ≥ target level, accuracy = 30% + (user_lvl − target_lvl); fails outright if target higher level |
| **Confusion self-hit** | (status effect) | 50% chance confused mon hits itself for a 40-power typeless physical |
| **Flinch** | Bite 10%, Stomp 30%, Headbutt 30%, Hyper Fang 10%, Low Kick 30%, Rolling Kick 30% | On hit, chance to cause target to skip their next action if it hasn't yet moved this turn |
| **Multi-hit** | Double Slap, Fury Attack, Pin Missile, Comet Punch, Bonemerang | Gen 1 distribution: 2 (37.5%), 3 (37.5%), 4 (12.5%), 5 (12.5%) |
| **Recoil** | Take Down, Double-Edge, Submission | User takes 25% of damage dealt back as recoil |
| **Crit-boost** | Slash, Razor Leaf, Karate Chop, Crabhammer | High crit rate (Gen 1: 8× base crit) |

Events added to `battleStateMachine.ts`:

- `move_charging` (solar beam / dig turn 1)
- `semi_invulnerable_miss` (attacks against a dug/flown target)
- `bide_storing`, `bide_unleash`
- `counter_fired`
- `rollout_stack` (with current multiplier)
- `drain_heal`, `leech_seed_applied`, `leech_seed_tick`
- `flinched`
- `multi_hit`, `crit`, `recoil`
- `confusion_self_hit`
- `ohko`, `ohko_failed`

**Shipped:**
- New optional flags on `MoveData`: `drainRatio`, `leechSeed`, `bide`, `rollout`, `counter`, `semiInvulnerable`, `charge`, `ohko`, `flinchChance`, `multiHit`, `recoilRatio`, `highCrit`.
- New transient `BattleMoveState` on `BattlePokemon` for Bide damage, Rollout streak, Counter stored damage, charge/semi-invulnerable flags, Leech Seed seeder, confusion-turns-left, and flinched flag. Reset on battle start, battle end, and swap.
- New `StatusType` member `confuse` with 1-4 turn duration (Gen 1) — confused mons have a 50% self-hit chance each turn (typeless 40-power physical against own defense).
- `MoveData` updates in `src/data/moves.ts`: drain on Absorb / Mega Drain / Leech Life (0.5); Leech Seed is now a status move (power 0, CD 3); Dig / Fly are semi-invulnerable (untargetable on charge turn); Solar Beam / Sky Attack are charge moves; Horn Drill / Fissure are OHKO (power 0, accuracy 30); Bite 10% / Headbutt 30% / Stomp 30% / Low Kick 30% / Rolling Kick 30% flinch; Fury Attack / Comet Punch / Pin Missile / Double Kick / Twineedle / Bonemerang are multi-hit; Take Down / Double-Edge / Submission have 25% recoil; Slash / Razor Leaf / Karate Chop / Crabhammer / Cross Chop have high crit. Counter, Bide, Rollout added as new named moves. Confusion + Psybeam now have 10% confuse.
- `BattleStateMachine.resolveAttack` is fully rewritten to sequence: flinch check → status skip → confusion self-hit roll → forced-action override (charging / biding) → Bide path → Charge path → Semi-invulnerable path → Counter path → useMove → semi-invulnerable target miss → OHKO path → accuracy check → damage loop (multi-hit, rollout power doubling, crit) → counter/bide state storage on defender → drain → recoil → leech seed → flinch set → secondary status effect. End-of-turn also runs Leech Seed ticks for both sides and decrements confusion duration.
- `damageCalc.ts` gains a Gen-1-style crit roll (`spd/512` base, `×8` for `highCrit` moves) and a `powerMultiplier` option used by Rollout.
- `BattleScene` handles every new event with HUD messages; `BattleHUD.STATUS_LABELS` gains `confuse: "CNF"`.
- Tests in `tests/battleStateMachine.test.ts` cover drain, recoil, OHKO hit + level-gate fail, semi-invulnerable miss, leech seed application + tick, and multi-hit event emission.

All screens using the Pokemon-list or Item-list pattern should behave identically. A **shared filter/tab component** is the foundation — build it once, use everywhere.

- Includes a **Reset all filters** button.
- Triggered from a "Type" pill button in any list screen.
- Reused in: party screen, Pokemon trader, Train screen, and (where applicable) Pokedex.
- **Shipped:** `src/ui/FilterModal.ts` exports `openFilterModal(scene, currentType, onSelect)`. ### 2.9 Train list sort prioritising evolutions + new moves (TODO #20) ✅

- Default Train screen sort:
  1. Pokemon with an evolution available **right now** (level/stone met + player has stone)
  2. Pokemon with a newly-available move (levelled up past an unlock)
  3. Then: ascending by level
- **Shipped:** New `getTrainSortedRoster()` on `MainMenuScene` computes a three-bucket rank via `getEvolutionInfo(p)?.canEvolve`, then `countNewMoves(p) > 0`, then level-ascending tie-break. Favourites still bubble to the top within each bucket so a starred Pokemon isn't buried under an unstarred evolution candidate. Type filter is applied first so the ranking always operates on the visible set. Card stroke colour now reflects the bucket (orange for evolve-ready, green for new-moves, neutral otherwise) so the priority is visible at a glance without needing to read the badge text.

## Bucket 3 — Dungeon flow & rewards

### 3.1 Area-cleared reward screen (TODO #6) ✅

- New `src/scenes/RoomClearScene.ts` (or modal in `MapScene`) triggered when the player steps onto the exit tile after clearing a room.
- Layout mirrors the Pokemon end-of-battle screen: **Area Cleared!** → XP gained → gold gained → any drops.
- **Drop table** (regular rooms):
  - 25% basic item — pooled from: Potion, Antidote, Paralyze Heal, Awakening, Repel, Escape Rope, X-Attack, X-Defense, X-Speed
  - 5% decent item — pooled from: Super Potion, Revive, Ether, Hyper Potion, random vitamin (Carbos/Iron/etc.)
  - 1% **Rare Egg**
  - 1% **Legendary Egg** — gated to World 5+
- Rolls are independent; a room can drop zero, one, or multiple rewards.
- Trainer/boss/gym rooms get a **guaranteed** decent item + egg chance (see 3.2).
- **Shipped:** new `src/scenes/RoomClearScene.ts` + `src/data/rewardDrops.ts` with per-pool roll helpers and scripted-RNG tests. Regular room exit tile in `MapScene.completeMap` now hands off to the scene instead of the old inline text overlay. Four new medicine items added to fill the drop pool — **Antidote**, **Paralyze Heal**, **Awakening** (status cures routed through a new `applyStatusCure` + `cure` `ApplyResult`) and **Hyper Potion** (+120 HP). Ether was intentionally dropped from the decent pool — the game uses cooldowns, not PP, so it has no mechanic. All four new medicines are stocked in PokeMart at world gates (cures from world 1, Hyper Potion from world 4). Legendary Egg gate confirmed at World 5+ per the PRD's original proposal.

### 3.2 Trainer-defeat screen + exit-tile fix (TODO #13) ✅

- Today: beating a boss turns the red square green, player walks off, but a wild encounter on the way out can reset it — player is forced to refight. Broken.
- **Fix:** on trainer/boss/gym victory, instead of green-tile-walk-out, show a **Trainer Defeat** screen (same component as 3.1 but themed): defeated-trainer portrait, "You defeated {Trainer}!", XP, gold, **guaranteed decent item + 1% egg roll**.
- From that screen → the room is marked cleared and the player transitions straight to the next room / world-select / town (same flow that the current green-tile handoff intends).
- No more returnable red-tile state. Kills the regression class entirely.
- **Shipped:** `BattleScene.handleBattleEnd` on boss win skips the VICTORY overlay and routes directly to `RoomClearScene` in trainer mode (scene kind `"trainer"`), carrying battle XP total, gold earned, level-ups, new-seen flag, and boss species id for the portrait. The scene uses `rollTrainerDrops` — guaranteed decent item + 1% rare/legendary egg rolls — and advances room progression on dismiss. `MapScene` no longer tracks `bossDefeated`; the exit tile stays red with `!` for the whole visit since the player never re-enters the map after a boss win (the red→green tile state is fully removed, regression class gone). Until Bucket 4 ships trainer/gym data, the "trainer" framing applies to the existing boss rooms at 5/10/15/20/25; those rooms will be replaced by proper trainer + gym content in 4.2/4.3.
