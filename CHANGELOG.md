# Changelog

All notable game changes are recorded here. Newest entries at the top. See `CLAUDE.md` → *Changelog Policy* for the format and rules.

## 2026-04-13

- **content:** **Two starter TMs** — `TM Headbutt` (Normal, Pow 45, CD 1, 250g) and `TM Body Slam` (Normal, Pow 55, CD 2, 350g) are now sold from the very first town visit (`worldRequired: 0`), giving players an immediate way to test the Training Centre TM flow without grinding to world 2. Like all TMs they're type-matched, so a Normal-type pokemon (Rattata, Pidgey, Eevee, Jigglypuff, Meowth, etc.) is required to actually learn them.
- **ui:** **Roster & Items screens gain sort + favourites** — the Pokemon screen now has a `LEVEL / TYPE / A-Z` sort bar at the top, and every roster card has a tappable ☆/★ star at its top-right corner. Starred pokemon are pinned to the top of the list regardless of the active sort, so your mains are always one tap away. Items gets a `A-Z / QTY` sort bar that re-orders each category section (Medicine, Battle, Vitamins, etc.) by name or quantity while preserving the category grouping. Favourite state persists across sessions via a new optional `fav` field on the saved pokemon record (old saves load cleanly with everyone unstarred). Touched `BattlePokemon.isFavourite`, `createBattlePokemon`, `saveManager` serialize/deserialize, and `MainMenuScene.drawRoster`/`drawItems`. Roster card tightened once more (h=58, spacing=64) to make room for the sort bar without losing the 10-per-page pagination from yesterday.

## 2026-04-12

- **ui:** **Long roster lists now paginate** — Pokemon, Select Party, Train, and the item-use target picker cap at 10 cards per page with a `< PREV` / Page X/Y / `NEXT >` strip above the BACK button. Previously the roster screen stopped drawing at 9 cards, Select Party at 12, and Train at 11 because each list did a hard `y > GAME_H - N` early-return instead of paging. Shared `drawPagination` / `pagedSlice` / `clampListPage` helpers on `MainMenuScene` keep the behaviour consistent, and the roster card was tightened from h=68/spacing=75 to h=60/spacing=66 so 10 fit comfortably above the pagination strip. `showScreen(screen, resetPage?)` and `showItemUseTarget(belt, resetPage?)` both take an optional flag so PREV/NEXT and party-toggle redraws don't snap back to page 1.
- **audio/ui:** Added **purchase/heal/item-use confirmation** — shops, PokeCenter, move-learning, vitamin/stone/potion use, map reveal, repel, and egg hatches now fire a short procedural SFX and a fading toast near the bottom of the screen. New `playSFX(kind)` helper on `MusicManager` with presets for `purchase` (ca-ching), `heal` (C-E-G chime), `item_use`, `hatch` (major arpeggio), `learn`, and `error`. Also added a reusable `src/ui/Toast.ts` component and `src/ui/itemFeedback.ts` that maps `applyItem` results to human-readable messages ("Charizard restored 30 HP!", "Grew to Lv14!", etc.) so MainMenu and MapScene share the same confirmation vocabulary. Evolution animations also play the celebration chime. Previously every purchase / field-item use silently refreshed the screen with no feedback.
- **content:** **Big item expansion** — the PokeMart catalogue grows from 6 to 34 items. Five **Evolution Stones** (Fire/Water/Thunder/Leaf/Moon, 500–600g, world 2–3) wire up the previously unreachable stone-line evolutions — Raichu, Ninetales, Arcanine, Vileplume, Victreebel, Exeggutor, Poliwrath, Cloyster, Starmie, Clefable, Wigglytuff, Nidoking, Nidoqueen, and all three Eeveelutions (Vaporeon/Jolteon/Flareon). Five **Vitamins** (HP Up, Protein, Iron, Carbos, Calcium, 400g, world 1) give permanent stat bonuses with caps of +25 HP / +15 per other stat. **Rare Candy** (700g, world 2) levels up a pokemon but is capped at the highest level in your roster so it only catches stragglers up. Four **X Items** (X Attack/Defend/Speed/Special, 100–120g, world 1) give +1 stage (×1.5) battle boosts for the rest of the current battle. **12 TMs** (Thunderbolt, Flamethrower, Ice Beam, Earthquake, Psychic, Surf, Shadow Ball, Hyper Beam, Fire Blast, Hydro Pump, Blizzard, Thunder, 800–1500g, world 2–4) teach specific moves via the Training Centre.
- **content:** TM compatibility is **type-matched only** — a pokemon can learn a TM if it shares at least one type with the move (Charizard can learn TM Flamethrower but not TM Thunderbolt; Pikachu can learn TM Thunder but not TM Ice Beam). Enforced via `canUseTM` in `shop.ts` and surfaced in the Train-screen UI with disabled `Type mismatch` / `Already knows` labels.
- **ui:** **Items screen redesigned** with category section headers (Medicine · Battle · Vitamins · Stones · Candy · TMs · Field). USE button is category-aware: medicine/vitamin/stone/candy open the roster picker; battle items show "Only usable in battle" toast; TMs show "TMs can only be used at the Training Centre" toast; field items display a static FIELD label (escape rope is still used from the dungeon). The roster picker now filters targets correctly per category (stones check `getStoneEvolution`, vitamins check caps, rare candy checks roster-level cap, medicine keeps the existing revive/heal filter) and shows a reason label ("Full HP", "No effect", "At roster cap", "ATK maxed") on disabled cards.
- **ui:** **PokeMart redesigned** with category grouping (Medicine · Field · Battle · Vitamins · Candy · Stones · TMs). Compact 32-tall cards fit the expanded catalogue without scroll. World-gating preserved via existing `worldRequired` filter.
- **ui:** **Training Centre gains a Use TMs panel** below the "Available to Learn" section — lists owned TMs with a TEACH button. Teaching consumes one TM; if the pokemon already knows 4 moves, a "Which move should be forgotten?" picker appears first. TMs without a type match or for already-known moves show a disabled `Type mismatch` / `Already knows` label.
- **combat:** Battle items (X Attack/Defend/Speed/Special) add **temporary stage boosts** tracked in new `BattlePokemon.battleBoosts: StageBoosts`. Each stage = ×1.5 (Gen 1 single-stage model), capped at 2 stages. Boosts reset at the start of every battle on the player's roster (enemies never boost). `damageCalc.ts` applies the multiplier to attacker atk/spc and defender def/spc; `battleStateMachine.resolveTurn` applies it to speed for turn order. A small `ATK▲ DEF▲ …` indicator appears above the active player sprite whenever any boost is active, and a new `stat_boost` battle event is rendered as "Charizard's Attack rose!".
- **combat:** The battle items panel now **filters out field items** (escape rope / repel / map) that previously appeared in battle and silently failed when used. Bonus fix alongside the new category-aware filter.
- **progression:** Vitamins persist in a new `BattlePokemon.statBonuses: Stats` field, layered onto base stats via a new `applyStatBonuses` helper. Bonuses are preserved across level-ups (`addXP`), evolution (new shared `evolveIntoSpecies` helper now called by both Train-screen level evolution and Items-screen stone evolution), and save/load (`saveManager.ts` only writes the field when non-zero). Existing saves load cleanly with bonuses defaulting to zero.
- **content:** Added `evolvesFrom` + new `evolutionStone` field to ~16 orphaned species in `pokemon.ts` so stones can trigger them. The existing `buildEvolutionMap` in `shop.ts` still filters on `evolutionLevel`, so stone-only evolutions correctly stay invisible to the Train-screen level-evolution flow — stones are the only path to them.
- **refactor:** Extracted the Train-screen inline evolution logic into a shared `evolveIntoSpecies(pokemon, newSpecies)` helper in `statCalc.ts`. Previously the Train screen mutated the pokemon in-place with its own stat-recalc code; now both that code path and the new stone-evolution path in `items.ts` go through the same function, guaranteeing vitamin bonuses and HP ratio behave identically whichever way you evolve.
- **test:** Added `tests/itemsAndStats.test.ts` (24 tests) covering `applyStatBonuses`, vitamin persistence through level-ups, `evolveIntoSpecies` HP/bonus preservation, stone evolution lookup (including Eevee branching), every `applyItem` category path, Rare Candy roster-cap behaviour, X Item stage capping, TM type-match rule, and the rejection of TMs via `applyItem` (they must go through the Training Centre). Also added 4 new `damageCalc` tests for the stage-multiplier path.
- **content:** Added **7 new status-only moves** (power 0 with a status effect) — `sing` (normal, 55% sleep), `hypnosis` (psychic, 60% sleep), `spore` (grass, 85% sleep — strongest sleeper), `lovely_kiss` (normal, 75% sleep), `glare` (normal, 75% paralyze), `poison_powder` (grass, 75% poison), `poison_gas` (poison, 55% poison). All CD 3. Fills out the status toolkit beyond the existing `sleep_powder`/`stun_spore`/`thunder_wave`/`toxic`.
- **content:** Distributed the new status moves across the expected Pokemon level-up learnsets to match Gen 1 canon — **Sing** on Jigglypuff/Wigglytuff/Clefairy/Clefable; **Hypnosis** on Drowzee/Hypno/Gastly/Haunter/Gengar; **Spore** on Paras/Parasect; **Lovely Kiss** on Jynx; **Glare** on Ekans/Arbok; **Poison Powder** on Bulbasaur/Oddish/Bellsprout/Paras/Venonat/Exeggcute/Tangela lines and their evolutions; **Poison Gas** on Grimer/Muk/Koffing/Weezing. Also added missing `sleep_powder` to Bulbasaur, Venonat, Exeggcute, and Tangela lines, `stun_spore` to Bellsprout and Exeggutor lines, and `thunder_wave` to Electabuzz/Voltorb/Electrode. Enemy AI already picks status moves ~10% of the time (`wildAI.ts:26`), so wild Pokemon will now actually cast these.
- **bugfix:** The **in-battle "Use <item> on who?" target panel** (`BattleHUD.showItemTargetPanel`) was drawing each party member as a flat TYPE_COLORS rectangle instead of the Pokemon's sprite — the earlier 7080e0b fix for invisible sprites only covered the town and dungeon item modals, not the in-battle one. Now loads `species.spriteKey` (with `NEAREST` filter, full opacity regardless of `canTarget`) and falls back to the type-color rectangle only if the texture is missing.
- **bugfix:** The XP bar on the dungeon **party strip** (the mini cards at the top of the grid) appeared to never update — the bar was only 3px tall and its bg/fill had mismatched origins (bg centered at `xpY+2`, fill top-left at `xpY`), so the tiny width changes between battles were imperceptible and looked glued in place. Aligned bg and fill to the same `(0, 0)` origin, bumped height to 4px, and added a numeric `XP n/needed` label directly under the bar so progress is unambiguous mid-run. `MapScene.drawPartyStrip`.
- **ui:** Added a **POKEMON** button to the dungeon bottom row (alongside ITEMS and ESCAPE) that opens a full-roster modal matching the town's "Your Pokemon" view — shows level, HP `x/y`, **numeric XP `x/y`** with bar, stats line (ATK/DEF/SPD/SPC), and a green PARTY marker on the 3 active party members. Makes it possible to track XP gains mid-run without escaping to town. Implemented in `MapScene.showPokemonModal`; bottom buttons shrunk 120→110 wide to fit three across.
- **bugfix:** Pokemon sprites in the **"Use item on?" target modals** (both town Items view and dungeon Items modal) were being rendered at `alpha 0.3` whenever the card was non-targetable (`!fainted && !fullHP`), which on the dark card background made them effectively invisible — especially right after a PokeCenter heal or on the dev test save where every Pokemon starts at full HP. Sprites now render at full opacity regardless; disabled state is still conveyed by card color and text color. Also bumped the sprite size (32→40 in town, 36→44 in dungeon) so they're easier to identify.
- **ui:** The **dungeon Items modal** (`MapScene.showItemModal`) now renders the same egg progress cards as the town Items view — tier-colored border, 40×40 sprite, `walked / total` count, progress bar, and a "Ready to hatch!" / "Hatching as you walk" flavor line. Crucial because eggs tick down during dungeon movement, so this is exactly where the player wants to check progress. Handles the empty-items-but-has-eggs case too.
- **ui:** Flipped the egg step counter from "remaining / total" to "walked / total" in both the Items view and the Egg Shop inventory — reads more naturally as progress (e.g. `3 / 150` after three steps instead of `147 / 150`). Underlying `EggInstance.stepsRemaining` still ticks down, only the display is inverted.
- **ui:** Beefed up the **egg display in the Items view** — eggs now render as full 50-tall cards with a tier-colored border, 40×40 sprite, bold name, step count shown as `remaining / total`, and a **progress bar ticking toward hatch**. Header shows the egg count. "Ready to hatch!" flavor appears when `stepsRemaining === 0`. The old 28-tall row was easy to miss below a long item list.
- **ui:** Main-menu inventory summary now appends `🥚 x<count>` (or shows `🥚 Eggs xN` alone if the player only carries eggs), so an egg you just bought is visible without opening the Items screen.
- **build:** Added a **DEV: +5000g** button on every occupied save slot card in the title screen so an existing run can be topped up for egg testing without starting over. Backed by a new `addGoldToSlot()` helper in `saveManager.ts` that edits the raw save JSON directly. Slot card height grew from 150 → 180 to fit the extra row.

Injects an advanced state for testing the egg system without playing through early game: level-30 Charmeleon/Wartortle/Ivysaur party (+ L25 Pikachu & Pidgeotto in roster), 5000 gold, full item belt, worlds 0–2 unlocked, 1 common + 1 rare egg already in the bag. Implemented via `createTestGameState()` in `saveManager.ts`.

Includes `src/data/eggs.ts`, `EggInstance`/`EggTier` types, save/load support, town egg shop UI, and step-tick wiring in `MapScene`.
