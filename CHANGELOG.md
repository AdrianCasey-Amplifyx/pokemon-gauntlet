# Changelog

All notable game changes are recorded here. Newest entries at the top. See `CLAUDE.md` → *Changelog Policy* for the format and rules.

## 2026-04-12

- **ui:** Added a **POKEMON** button to the dungeon bottom row (alongside ITEMS and ESCAPE) that opens a full-roster modal matching the town's "Your Pokemon" view — shows level, HP `x/y`, **numeric XP `x/y`** with bar, stats line (ATK/DEF/SPD/SPC), and a green PARTY marker on the 3 active party members. Makes it possible to track XP gains mid-run without escaping to town. Implemented in `MapScene.showPokemonModal`; bottom buttons shrunk 120→110 wide to fit three across.
- **bugfix:** Pokemon sprites in the **"Use item on?" target modals** (both town Items view and dungeon Items modal) were being rendered at `alpha 0.3` whenever the card was non-targetable (`!fainted && !fullHP`), which on the dark card background made them effectively invisible — especially right after a PokeCenter heal or on the dev test save where every Pokemon starts at full HP. Sprites now render at full opacity regardless; disabled state is still conveyed by card color and text color. Also bumped the sprite size (32→40 in town, 36→44 in dungeon) so they're easier to identify.
- **ui:** The **dungeon Items modal** (`MapScene.showItemModal`) now renders the same egg progress cards as the town Items view — tier-colored border, 40×40 sprite, `walked / total` count, progress bar, and a "Ready to hatch!" / "Hatching as you walk" flavor line. Crucial because eggs tick down during dungeon movement, so this is exactly where the player wants to check progress. Handles the empty-items-but-has-eggs case too.
- **ui:** Flipped the egg step counter from "remaining / total" to "walked / total" in both the Items view and the Egg Shop inventory — reads more naturally as progress (e.g. `3 / 150` after three steps instead of `147 / 150`). Underlying `EggInstance.stepsRemaining` still ticks down, only the display is inverted.
- **ui:** Beefed up the **egg display in the Items view** — eggs now render as full 50-tall cards with a tier-colored border, 40×40 sprite, bold name, step count shown as `remaining / total`, and a **progress bar ticking toward hatch**. Header shows the egg count. "Ready to hatch!" flavor appears when `stepsRemaining === 0`. The old 28-tall row was easy to miss below a long item list.
- **ui:** Main-menu inventory summary now appends `🥚 x<count>` (or shows `🥚 Eggs xN` alone if the player only carries eggs), so an egg you just bought is visible without opening the Items screen.
- **build:** Added a **DEV: +5000g** button on every occupied save slot card in the title screen so an existing run can be topped up for egg testing without starting over. Backed by a new `addGoldToSlot()` helper in `saveManager.ts` that edits the raw save JSON directly. Slot card height grew from 150 → 180 to fit the extra row.
- **build:** Added a **DEV: TEST SAVE (5000g)** button on slot 1's empty card in the title screen. Injects an advanced state for testing the egg system without playing through early game: level-30 Charmeleon/Wartortle/Ivysaur party (+ L25 Pikachu & Pidgeotto in roster), 5000 gold, full item belt, worlds 0–2 unlocked, 1 common + 1 rare egg already in the bag. Implemented via `createTestGameState()` in `saveManager.ts`.
- **content:** Added the **egg shop & hatching system** — three tiers (Common 250g/50 steps, Rare 750g/150 steps, Legendary 2000g/400 steps), step-based hatching tied to dungeon movement, hatch level = avg level of top-3 roster (min 5). Includes `src/data/eggs.ts`, `EggInstance`/`EggTier` types, save/load support, town egg shop UI, and step-tick wiring in `MapScene`. (Code was on disk before the repo existed — this commit brings it into git history.)
- **test:** Added `tests/eggs.test.ts` covering egg tier rolls, hatch-level calculation, and step ticking.
- **docs:** Synced `CLAUDE.md` with current codebase — documented status effects (burn/poison/paralyze/sleep), boss rooms (5/10/15/20/25) with out-of-pool boss species, `Repel`/`Map` field items, the `seenPokemon` shop-unlock mechanic, and `repelSteps` in `GameState`.
- **docs:** Added Changelog Policy section to `CLAUDE.md` and created this file so future changes are logged with date + summary.
- **git:** Initialized git repo and pushed to `https://github.com/AdrianCasey-Amplifyx/pokemon-gauntlet` (public, branch `main`) with a `.gitignore` excluding `node_modules`, `dist`, `.env*`, and editor cruft.
- **docs:** Added `src/data/eggs.ts` + `GameState.eggs` to `CLAUDE.md` directory listing and data model (had been missed on the first sync).
- **docs:** Replaced the Changelog Policy with a broader **End-of-Task Workflow** in `CLAUDE.md` — now requires updating `CHANGELOG.md`, `GAME_DESIGN.md`, and `CLAUDE.md` (as applicable), then committing and pushing to `origin main` for every game change. Added a dedicated **Git & GitHub Policy** section covering commit style, co-author trailer, branch rules, and destructive-op safeguards.
- **docs:** Comprehensively rewrote `GAME_DESIGN.md` to describe the game as it actually ships — real combat formula (`/65`, not `/50`), 15-type chart, status effects (burn/poison/paralyze/sleep), 8 worlds × 25 rooms with scaling formulas, boss rooms at 5/10/15/20/25 drawing from an out-of-pool `WORLD_BOSSES` list, fog-of-war grid map generation, the town shop system (PokeCenter 20g, world-gated PokeMart, Buy Pokemon from `seenPokemon`, Train, Evolve), the egg system (common 250g/50 steps, rare 750g/150 steps, legendary 2000g/400 steps), current item list, XP/level/evolution curves, save behavior, and a clearly-marked Future Directions section for unbuilt ideas (catching, trainers, held items, TMs, research tasks, Lab upgrades).
