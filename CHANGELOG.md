# Changelog

All notable game changes are recorded here. Newest entries at the top. See `CLAUDE.md` → *Changelog Policy* for the format and rules.

## 2026-04-17

- **content:** TM compatibility rebased on Gen 1 Bulbapedia per-species learnsets (PRD 2026-04-17 §1.1). Previously TMs required a shared type between the pokemon and the move, which over-restricted cross-type canon compatibility (Clefairy couldn't learn Body Slam; Charmander couldn't learn Earthquake; etc). New `src/data/tmCompatibility.ts` carries the per-TM allow-list; `canUseTM` now returns `incompatible` / `Can't learn` instead of `type_mismatch` / `Type mismatch`. The broad cases match canon: Clefairy learns almost every TM, Body Slam is compatible with most species, and unsuitable species (Caterpie, Magikarp, etc.) are still correctly rejected.
- **combat:** Moves can now miss. Every move's accuracy is audited against Gen 1 Bulbapedia (PRD 2026-04-17 §1.3) — Horn Drill and Fissure are properly 30% OHKOs-in-progress, Hydro Pump drops to 80, Blizzard to 90, Sing to 55, Hypnosis to 60, Thunder to 70, Fire Blast to 85, etc. Under-100 accuracy moves now roll against `rng >= accuracy/100`; a miss emits `move_missed`, burns the cooldown, and the HUD surfaces "attack missed!". Previously every move was locked at 100% because the miss path didn't exist.
- **combat:** X-item boosts are now strictly one-battle-only (PRD 2026-04-17 §1.5). Boosts persist through switches as designed, but a pokemon's `battleBoosts` are cleared the moment it faints (so reviving mid-battle brings it back clean), and every player party member's boosts are wiped when the battle ends. Previously boosts could linger into the next battle on pokemon that weren't on the active party when the next `start()` ran. Implementation: new `clearBoosts` helper + `endBattle` method in `src/core/battleStateMachine.ts`; 4 new tests cover apply/swap/faint/battle-end transitions.
- **ui:** Train screen now blocks forgetting the last no-cooldown move (PRD 2026-04-17 §1.4). Both the `FORGET` button on Current Moves and the TM forget-picker enforce `isLastCdZero`, with a tap on the disabled button surfacing the reason via toast so players aren't left guessing why the action is disabled.

## 2026-04-15

- **bugfix:** **Modal click-through, take 4 — force-unregister input on every child before destroying.** Take 3 added interactive backdrops but they weren't enough on their own: when you hit BACK from a menu to the town hub and tapped in the void under the `ADVENTURE` button, a click still fired the button that used to sit at that position on the previous screen. Root cause: Phaser 3's input plugin has a one-tick lag when cleaning up destroyed interactive GameObjects — they linger in the hit-test list for a frame, so the next pointerdown can still dispatch to an old handler even after `this.children.removeAll(true)`. New `resetScreen` helper in `MainMenuScene` walks every child and calls `disableInteractive()` before `removeAll(true)` — that forces synchronous removal from the input plugin so the hit list is clean on the very next pointer event. Every screen rebuild in the scene (`buildUI`, `showScreen`, `showItemUseTarget`, `showEvolutionAnimation`, `drawTrainMoves`, `showTMForgetPicker`) now goes through it. Same pattern applied inline to `MapScene.buildMapUI`.
- **shop:** **Buy Pokemon now lists every species you've seen, even duplicates.** Previously the trader hid any species already in your roster, so if you caught a Lv5 Geodude you could never buy the Lv22 Geodude you'd later seen in world 3. `getAvailableShopPokemon` no longer filters by ownership — every entry in `seenPokemon` shows up, priced at the highest wild level you've encountered. Touched `src/data/shop.ts` and `src/scenes/MainMenuScene.ts`.
- **bugfix:** **Modal click-through, take 3 — Pokedex and other `MainMenuScene` screens were leaking taps to destroyed buttons underneath.** The Pokedex (and roster / items / shop / trader / egg shop / party / train screens) are drawn by `showScreen()`, which calls `this.children.removeAll(true)` before building the new screen. Even though that destroys the old buttons, Phaser 3's input plugin holds a stale reference to a destroyed interactive object for at least a frame — so if a non-interactive element on the new screen happened to sit where a freshly-destroyed button used to be, a tap on that spot would still fire the old button's handler. In the Pokedex's case, the 8th card (y 534–594) sat right on top of where the main menu's `ADVENTURE` button used to be (y 508–556), so tapping a card navigated to the party picker — matching the user-reported "clicks right through to the next menu" symptom exactly. Fix: every full-screen replacement in `MainMenuScene` (`buildUI`, `showScreen`, `showItemUseTarget`, `showEvolutionAnimation`, `drawTrainMoves`, `showTMForgetPicker`) now calls `.setInteractive()` on its fresh dark backdrop rectangle, so the backdrop is the top-most interactive object and absorbs any empty-space click before any stale registration can fire.
- **bugfix:** **Modal click-through, take 2 — the real culprit was MapScene's scene-level tap-to-move listener.** The first pass added `setInteractive()` backdrops to the BattleHUD overlays, which is necessary but not sufficient — `MapScene` registers a `this.input.on("pointerdown", ...)` handler for tap-to-move, and scene-level input listeners fire on every pointer event regardless of which GameObject is on top. So opening the POKEMON / ITEMS / TARGET modals and tapping the dim backdrop would silently move the player behind the modal (and potentially trigger encounters). Added an `activeModals` counter + `registerModal()` helper in `MapScene`; the tap-to-move handler early-returns while any modal is open, and every modal (`showPokemonModal`, `showItemModal`, `showItemTargetModal`, `playHatchOverlay`, the room-clear overlay) registers itself so it auto-decrements on `destroy`. Also fixed a latent bug where `buildMapUI` was called multiple times per run (after using Map / Repel, etc.) and stacked duplicate pointerdown listeners — now calls `removeAllListeners("pointerdown")` before registering. As belt-and-suspenders on the battle side, `BattleHUD` now also explicitly `disableInteractive()`s the move buttons and SWAP/ITEMS actions while any overlay is open, and only re-enables visible slots on close — so even if Phaser's topOnly hit-test had a gap, clicks can't reach the HUD controls.
- **bugfix:** **SELECT PARTY first card overlapped the sort bar.** The `LEVEL / TYPE / A-Z` bar sat at y=96 (extending to 107) but the first Pokemon card was at y=120 with height 46, so its top edge at 97 clipped 10px into the sort buttons. Pushed the sort bar to y=104 (clear of the preview sprite strip at y=72) and moved the first card to y=140 so it sits comfortably below the sort bar's bottom at 115.
- **content:** **Pokedex added to the town hub.** A new `POKEDEX` button opens a paged (10/page) catalogue of all 151 Gen 1 species in national-dex order with three reveal tiers: **unseen** slots show a dim `?` tile and `???` name; **seen** slots (encountered in battle but never owned) reveal the sprite + name in a muted tone with a `SEEN` badge and the hint "Catch this Pokemon to reveal its details"; **caught** slots (starter / purchased / hatched — tracked persistently so selling a Pokemon doesn't revoke its dex entry) show a `✓ CAUGHT` badge, full types, rarity label, and a short one-sentence flavor description. Header shows running totals `Seen X/151   Caught Y/151`. New state field `caughtPokemon: string[]` on `GameState`, auto-inferred from current roster for legacy saves, and updated at every capture point (`StarterSelectScene`, `MainMenuScene.drawPokemonBuyList`, `MapScene.tickEggsAndMaybeHatch`). New content file `src/data/pokedexEntries.ts` carries flavor text for every species. Save format bumped to version 3 with the new `caughtPokemon` field (optional — legacy saves load cleanly and back-fill from the roster). Town hub gap tightened slightly to fit the 9th button without overlapping the TITLE SCREEN button.
- **ui:** **Adventure party picker now has sort filters** — the `SELECT PARTY` screen got the same `LEVEL / TYPE / A-Z` sort bar the Pokemon roster screen has, sharing the `rosterSort` state so you don't have to re-pick your preferred order twice. Favourites are still pinned to the top, and toggling a selection no longer resets the page. Each party card also now shows type codes alongside the level and stars favourited mons so you can eyeball coverage at a glance.
- **shop:** **Buy Pokemon now sells at the highest level you've seen in the wild.** Previously every shop Pokemon was a flat level 5 regardless of how tough the wild version was; now `seenPokemon` tracks speciesId → highest-ever encountered level (replacing the old `string[]`), and `getPokemonPurchaseCost` scales base cost linearly from level 5 (lvl 5 = 1x, lvl 10 = 2x, lvl 20 = 4x). Saves with the legacy array shape migrate on load to a `Record<string, number>` defaulting every seen species to level 5. Touched `types.ts`, `StarterSelectScene`, `BattleScene`, `MapScene` (egg hatch), `saveManager`, `data/shop.ts`, `MainMenuScene.drawPokemonBuyList`.
- **shop:** **Sell excess Pokemon for half the current buy price.** `POKEMON TRADER` (renamed from `BUY POKEMON`) now has a `BUY / SELL` tab switcher. The sell tab lists your roster with a `+Ng` sell value per card (`getPokemonSellValue` = purchase cost / 2, so it scales with level), blocks favourites to prevent accidents, hides the last Pokemon so you can't brick your save, and shows a confirm-sell overlay with the Pokemon's sprite before committing. Selling also clears any stale `selectedParty` indices so the adventure picker doesn't reference a removed mon.
- **docs:** Added `TODO.md` with the active feature backlog (party picker filters, seen-level shop pricing, sell Pokemon) and ticked off the three items above.

## 2026-04-13

- **bugfix:** **World progression got stuck at "Room 26/25" after clearing the final room of a world** — `MapScene.completeMap` incremented `world.currentMap` and unlocked the next world but never advanced `gameState.activeWorld`, so the town hub kept reading room `currentMap + 1 = 26` for the cleared world instead of room 1 of the next world. Now the final-room clear both unlocks and auto-advances `activeWorld` (when a next world exists), and the completion overlay reads `"Advancing to <next world name>!"`. `saveManager.buildGameState` also heals pre-fix saves on load by detecting `worlds[activeWorld].currentMap >= MAPS_PER_WORLD` and advancing, so anyone already stuck self-heals next session.
- **bugfix:** **4th move button was inert after switching pokemon** — `BattleHUD.create()` only wired each move button's click handler inside the `i < playerPokemon.moves.length` guard, so if the lead party member had fewer than 4 moves the slot-3 button was created with no `onClick`. Switching mid-battle to a pokemon with 4 moves (e.g. Butterfree with Gust in the 4th slot) relabelled the button via `updateMoveButtons` but never wired the callback, making the 4th move unusable. Now every slot gets its `onClick` at create time unconditionally; unfilled slots are simply `setVisible(false)` so they can't fire. Affected any pokemon whose 4th move was visible on the HUD after a swap.
- **content:** **Two starter TMs** — `TM Headbutt` (Normal, Pow 45, CD 1, 250g) and `TM Body Slam` (Normal, Pow 55, CD 2, 350g) are now sold from the very first town visit (`worldRequired: 0`), giving players an immediate way to test the Training Centre TM flow without grinding to world 2. Like all TMs they're type-matched, so a Normal-type pokemon (Rattata, Pidgey, Eevee, Jigglypuff, Meowth, etc.) is required to actually learn them.
- **ui:** **Roster & Items screens gain sort + favourites** — the Pokemon screen now has a `LEVEL / TYPE / A-Z` sort bar at the top, and every roster card has a tappable ☆/★ star at its top-right corner. Starred pokemon are pinned to the top of the list regardless of the active sort, so your mains are always one tap away. Items gets a `A-Z / QTY` sort bar that re-orders each category section (Medicine, Battle, Vitamins, etc.) by name or quantity while preserving the category grouping. Favourite state persists across sessions via a new optional `fav` field on the saved pokemon record (old saves load cleanly with everyone unstarred). Touched `BattlePokemon.isFavourite`, `createBattlePokemon`, `saveManager` serialize/deserialize, and `MainMenuScene.drawRoster`/`drawItems`. Roster card tightened once more (h=58, spacing=64) to make room for the sort bar without losing the 10-per-page pagination from yesterday.

## 2026-04-12

- **ui:** **Long roster lists now paginate** — Pokemon, Select Party, Train, and the item-use target picker cap at 10 cards per page with a `< PREV` / Page X/Y / `NEXT >` strip above the BACK button. Previously the roster screen stopped drawing at 9 cards, Select Party at 12, and Train at 11 because each list did a hard `y > GAME_H - N` early-return instead of paging. Shared `drawPagination` / `pagedSlice` / `clampListPage` helpers on `MainMenuScene` keep the behaviour consistent, and the roster card was tightened from h=68/spacing=75 to h=60/spacing=66 so 10 fit comfortably above the pagination strip. `showScreen(screen, resetPage?)` and `showItemUseTarget(belt, resetPage?)` both take an optional flag so PREV/NEXT and party-toggle redraws don't snap back to page 1.
- **audio/ui:** Added **purchase/heal/item-use confirmation** — shops, PokeCenter, move-learning, vitamin/stone/potion use, map reveal, repel, and egg hatches now fire a short procedural SFX and a fading toast near the bottom of the screen. New `playSFX(kind)` helper on `MusicManager` with presets for `purchase` (ca-ching), `heal` (C-E-G chime), `item_use`, `hatch` (major arpeggio), `learn`, and `error`. Also added a reusable `src/ui/Toast.ts` component and `src/ui/itemFeedback.ts` that maps `applyItem` results to human-readable messages ("Charizard restored 30 HP!", "Grew to Lv14!", etc.) so MainMenu and MapScene share the same confirmation vocabulary. Evolution animations also play the celebration chime. Previously every purchase / field-item use silently refreshed the screen with no feedback.
- **content:** **Big item expansion** — the PokeMart catalogue grows from 6 to 34 items. Five **Evolution Stones** (Fire/Water/Thunder/Leaf/Moon, 500–600g, world 2–3) wire up the previously unreachable stone-line evolutions — Raichu, Ninetales, Arcanine, Vileplume, Victreebel, Exeggutor, Poliwrath, Cloyster, Starmie, Clefable, Wigglytuff, Nidoking, Nidoqueen, and all three Eeveelutions (Vaporeon/Jolteon/Flareon). Five **Vitamins** (HP Up, Protein, Iron, Carbos, Calcium, 400g, world 1) give permanent stat bonuses with caps of +25 HP / +15 per other stat. **Rare Candy** (700g, world 2) levels up a pokemon but is capped at the highest level in your roster so it only catches stragglers up. Four **X Items** (X Attack/Defend/Speed/Special, 100–120g, world 1) give +1 stage (×1.5) battle boosts for the rest of the current battle. **12 TMs** (Thunderbolt, Flamethrower, Ice Beam, Earthquake, Psychic, Surf, Shadow Ball, Hyper Beam, Fire Blast, Hydro Pump, Blizzard, Thunder, 800–1500g, world 2–4) teach specific moves via the Training Centre.
- **content:** TM compatibility is **type-matched only** — a pokemon can learn a TM if it shares at least one type with the move (Charizard can learn TM Flamethrower but not TM Thunderbolt; Pikachu can learn TM Thunder but not TM Ice Beam). Enforced via `canUseTM` in `shop.ts` and surfaced in the Train-screen UI with disabled `Type mismatch` / `Already knows` labels.
- **ui:** **Items screen redesigned** with category section headers (Medicine · Battle · Vitamins · Stones · Candy · TMs · Field). - **ui:** **PokeMart redesigned** with category grouping (Medicine · Field · Battle · Vitamins · Candy · Stones · TMs). Compact 32-tall cards fit the expanded catalogue without scroll. World-gating preserved via existing `worldRequired` filter.

Each stage = ×1.5 (Gen 1 single-stage model), capped at 2 stages. Boosts reset at the start of every battle on the player's roster (enemies never boost). `damageCalc.ts` applies the multiplier to attacker atk/spc and defender def/spc; `battleStateMachine.resolveTurn` applies it to speed for turn order. A small `ATK▲ DEF▲ …` indicator appears above the active player sprite whenever any boost is active, and a new `stat_boost` battle event is rendered as "Charizard's Attack rose!".
